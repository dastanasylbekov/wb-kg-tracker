# Трекер изменений справочного центра Wildberries (Кыргызстан)

Отслеживает дату «Обновлено DD.MM.YYYY» у статей раздела
`https://seller.wildberries.ru/instructions/ru/kg/` и показывает, у каких статей
изменилась дата (а значит — внесли правки).

## Как это работает

1. `wb_kg_tracker.py` обходит весь раздел `/ru/kg/` (categories → category →
   subcategory → material) и собирает по каждой статье: заголовок, дату
   «Обновлено», признак «для Кыргызстана».
2. Результат сохраняется в **`snapshot.json`** (отсортирован по URL).
3. Скрипт сравнивает новый снимок с прошлым и пишет человекочитаемый отчёт
   **`CHANGES.md`**: новые статьи, сменившие дату, исчезнувшие.
4. На GitHub снимок **коммитится после каждого прогона**, поэтому
   `git log -p snapshot.json` — это полный журнал: видно, что и когда менялось,
   навсегда.

Сайт отдаётся сервером уже отрендеренным (SSR), поэтому headless-браузер
не нужен — хватает `requests`.

## Три способа запуска (всё одним механизмом)

- **По будням** — cron в `.github/workflows/track-wb-kg.yml`
  (`0 3 * * 1-5` = 09:00 по Бишкеку).
- **По расписанию** — поменяйте строку `cron` под себя.
- **Вручную** — вкладка *Actions* → этот workflow → *Run workflow*.

## Установка

1. Создайте репозиторий (приватный — нормально) и положите туда:
   - `wb_kg_tracker.py`
   - `.github/workflows/track-wb-kg.yml`  ← файл `track-wb-kg.yml` нужно положить
     именно в папку `.github/workflows/`
2. (Опционально) Telegram-уведомления: *Settings → Secrets and variables →
   Actions* → добавьте `TG_TOKEN` (токен бота от @BotFather) и `TG_CHAT`
   (ваш chat id). Без них уведомление просто пропускается.
3. Первый запуск сделайте вручную — он создаст исходный `snapshot.json`
   (в нём все статьи будут помечены как «новые», это нормально). Дальше каждый
   прогон показывает только разницу.

## Локальный запуск

```bash
pip install requests beautifulsoup4 lxml
python wb_kg_tracker.py            # обойти, обновить снимок, показать изменения
python wb_kg_tracker.py --notify   # + Telegram (env TG_TOKEN, TG_CHAT)
python wb_kg_tracker.py --delay 1  # вежливее к серверу: пауза 1 сек между запросами
```

## Если у части статей не распознаётся дата

Скрипт предупредит: `дата не распознана у N статей`. Это значит, что у этих
страниц «Обновлено» лежит в другом месте/формате. Решение — посмотреть исходный
HTML такой страницы (`curl` или DevTools → вкладка «исходный код страницы») и при
необходимости поправить `DATE_RE` в начале `wb_kg_tracker.py`. Текущая регулярка
ловит варианты вида `Обновлено 18.05.2026` и `Обновлена 1.6.2026`.

## Заметки

- Признак `kg: true/false` отделяет статьи именно для Кыргызстана от
  универсальных (в универсальных нет фразы «для продавцов из Кыргызстана»).
- Если Wildberries когда-нибудь перенесёт дату в контент, подгружаемый скриптом
  на клиенте, обычного `requests` станет мало — тогда обход той же логикой
  переносится на Playwright. Сейчас этого не требуется.
